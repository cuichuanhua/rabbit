1. Box 文件存储的是什么数据？
Box 文件是一种文本格式的训练标注文件，每一行包含以下信息：
字符 左边界 下边界 右边界 上边界 页码
你 50 100 80 130 0
好 85 100 115 130 0
世 120 100 150 130 0
界 155 100 185 130 0
具体含义：
字符：正确的文字内容（ground truth）
左/下/右/上边界：该字符在图片中的像素坐标位置
页码：通常是 0（单页训练时）

2. 为什么微调和训练需要生成 Box 文件？
核心原因：监督学习需要"正确答案"
Tesseract 的 LSTM 模型训练是有监督学习，需要告诉模型：
✅ 图片中有什么字符（正确文本）
✅ 每个字符在哪里（位置标注）
Box 文件就是这种标注数据的载体。

如何生成box 文件
1：让 Tesseract 自己先识别一遍，自动生成初始标注
    流程：
        把图片给 Tesseract
        Tesseract 识别出文字内容
        Tesseract 同时记录每个字符的位置坐标
        输出成 box 格式文件
    适用场景：
        大批量图片初步标注
        图片质量较好、字体规范的情况
        作为后续人工修正的基础

2：基于已知正确答案的反向匹配
核心思想： 我已经知道图片里应该是什么文字，只需要把文字和位置对应起来
流程：
    准备图片和对应的正确文本（ground truth）
    用 OCR 识别图片，获取每个字符的位置坐标
    用正确文本替换掉 OCR 识别错误的字符
    保留位置信息，生成修正后的 box 文件

box 文件只是中间产物，最终目标是生成 LSTM 训练需要的 .lstmf 文件

场景 1：生成训练数据
# 从图片和正确文本生成 box 文件
tesseract sample.png sample --psm 6 batch.nochop makebox
这会生成一个初始的 box 文件，但可能包含 OCR 识别错误

场景 2：修正错误样本（您的方案重点）

为什么要修正？
Tesseract 初次识别可能有误（如把 "0" 识别成 "O"）
训练时必须使用完全正确的标注
修正后的 box 文件才能用于高质量微调

场景 3：转换为 LSTM 训练格式
# 将 box 文件转换为 LSTM 训练需要的 .lstmf 格式
tesseract sample.png sample lstm.train

这会生成 sample.lstmf 文件，内部包含了 box 文件的标注信息。
场景 4：可视化验证
可以用工具查看 box 文件标注是否正确：
在图片上画出每个字符的边界框
显示对应的字符标签
人工检查标注质量
