OCR 引擎模式 - tessedit_ocr_engine_mode - 默认值: 3 - 场景: 0=仅传统, 1=仅 LSTM, 2=两者, 3=默认，效果提升⭐⭐⭐⭐⭐
页面分割模式 - tessedit_pageseg_mode - 默认值: 3 - 场景: 控制页面布局和文本方向，效果提升⭐⭐⭐⭐⭐
输出 HOCR - tessedit_create_hocr - 默认值: 0 - 场景: 生成带位置和置信度的 HTML，效果提升⭐⭐⭐
输出 TSV - tessedit_create_tsv - 默认值: 0 - 场景: 生成结构化表格数据，效果提升⭐⭐⭐
降噪级别 - denoise_level - 默认值: 0 - 场景: 嘈杂图像设置 1-3，效果提升⭐⭐⭐
对比度增强 - contrast_enhance - 默认值: 0 - 场景: 低对比度图像启用，效果提升⭐⭐⭐
自适应阈值块大小 - adaptive_threshold_blocksize - 默认值: 15 - 场景: 光照不均图像调整，效果提升⭐⭐⭐
强制比例单词 - textord_force_make_prop_words - 默认值: 0 - 场景: 西文文档分词优化，效果提升⭐⭐
启用公式检测 - equationdetect_enable - 默认值: 1 - 场景: 数学文档禁用可提速，效果提升⭐⭐
LSTM 网络模式 - lstm_network_mode - 默认值: 1 - 场景: 选择最适合的网络架构，效果提升⭐⭐

参数优先级规则 - 说明: 命令行参数 > -c 配置 > 配置文件 > 默认值 - 示例: --psm 6 等同于 -c tessedit_pageseg_mode=6
多个 -c 参数的使用 - 说明: 可以多次使用 -c 分别设置不同参数 - 示例: tesseract image.png output -c param1=val1 -c param2=val2
使用配置文件 - 说明: 将多个参数写入文件，每行一个参数 - 示例: 创建 config.conf 文件，使用时 tesseract image.png output config.conf
查看所有参数 - 说明: 使用 --print-parameters 命令输出当前版本所有可用参数 - 示例: tesseract --print-parameters > params.txt

📊 参数分类速查表
【性能优化类】
- lstm_use_matrix=1 (启用矩阵加速)
- language_model_ngram_use_suffix_array=1 (大数据集加速)
- pitsync_linear_version=6 (快速间距算法)
- textord_fast_pitch_test=1 (更快间距测试)

【精度提升类】
- lstm_choice_iterations=3-5 (增加迭代)
- beam_width=15-20 (扩大搜索)
- language_model_ngram_order=4-5 (增加上下文)
- word_dict_score_weight=0.8 (强化词典)

【调试诊断类】
- textord_debug_tabfind=1 (表格调试)
- textord_show_blocks=1 (显示文本块)
- classify_debug=1 (分类器调试)
- lstm_debug=1 (LSTM 调试)

【特殊场景类】
- tessedit_char_whitelist (字符限制)
- classify_bln_numeric_mode=1 (纯数字)
- textord_tabfind_vertical_text=1 (竖排文本)
- equationdetect_enable=0 (禁用公式检测)

【输出格式类】
- tessedit_create_hocr=1 (HTML 输出)
- tessedit_create_tsv=1 (TSV 输出)
- tessedit_create_alto=1 (ALTO XML)
- tessedit_create_pdf=1 (PDF 输出)

⚠️ 注意事项
1. 参数兼容性：不同 Tesseract 版本参数可能不同，4.x 和 5.x 有差异
2. 性能权衡：提高精度通常会降低速度，需根据场景平衡
3. 语言依赖：某些参数对中文效果明显，对英文可能不明显
4. 图像质量：高质量图像不需要过多参数调整
5. 测试验证：生产环境使用前务必充分测试
6. 日志监控：启用调试参数会产生大量输出，仅用于开发
7. 内存占用：增加 beam_width 和 ngram_order 会增加内存使用
8. 训练数据：参数效果依赖于训练数据质量

问题 1：识别率低

- 增加 beam_width=20
- 调整 user_defined_dpi=300

- 减少 lstm_choice_iterations=1
- 减少 beam_width=5
- 设置 language_model_ngram_on=0
- 禁用 equationdetect_enable=0

问题 3：误识别特殊字符
解决方案：
- 设置 tessedit_char_blacklist 排除不需要的字符
- 设置 tessedit_char_whitelist 只保留需要的字符
- 增加 language_model_penalty_chartype=0.6

问题 4：表格识别混乱
解决方案：
- 确认 textord_tabfind_find_tables=1
- 确认 textord_table_recognition=1
- 设置 tessedit_preserve_blanks=1
- 增加 user_defined_dpi=400

问题 5：小字体识别差
解决方案：
- 降低 textord_min_xheight=5
- 降低 textord_min_linesize=1.5
- 增加 user_defined_dpi=600
- 启用 denoise_level=1

问题 6：倾斜文档识别错误
解决方案：
- 启用 textord_biased_skewcalc=1
- 启用 textord_interpolating_skew=1
- 增加 textord_skewsmooth_offset=6
- 设置 textord_parallel_baselines=1

问题 7：噪声干扰严重
解决方案：
- 增加 denoise_level=2 或 3
- 启用 textord_heavy_nr=1
- 设置 textord_noise_rejwords=1

问题 8：中文竖排文本识别失败

- 启用 textord_tabfind_vertical_text=1

- 增加 classify_integer_matcher_multiplier=6

问题 9：黑白反转图像识别错误

- 禁用 tessedit_do_invert=0
- 或启用 tessedit_do_invert=1 自动检测
- 调整 adaptive_threshold_blocksize=25
- 增加 contrast_enhance=1

问题 10：多语言混合识别差

- 使用 tessedit_load_sublangs 加载子语言
- 增加 language_model_penalty_script=0.7

技巧 3：分页模式选择
- PSM 3：通用文档（默认）
- PSM 6：统一文本块（推荐）
- PSM 7：单行文本
- PSM 8：单个单词
- PSM 10：单个字符
- PSM 11：稀疏文本
- PSM 12：带 OSD 的稀疏文本

技巧 4：OCR 引擎选择
- OEM 0：仅传统引擎（速度快，准确率低）
- OEM 1：仅 LSTM 引擎（推荐，平衡）

- OEM 3：自动选择（默认）

技巧 5：输出格式选择
- TXT：纯文本，最小体积
- HOCR：HTML + 位置信息，适合网页展示
- TSV：结构化数据，适合表格提取
- PDF：可搜索 PDF，适合文档归档
- ALTO：XML 标准格式，适合数字图书馆

import net.sourceforge.tess4j.Tesseract;
import java.io.File;

public class TesseractExample {
    public static void main(String[] args) {
        Tesseract tesseract = new Tesseract();
        
        // 设置语言
        tesseract.setLanguage("chi_sim+eng");

        tesseract.setTessVariable("lstm_choice_iterations", "3");
        tesseract.setTessVariable("beam_width", "15");
        tesseract.setTessVariable("word_dict_score_weight", "0.8");
        
        try {

        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

🌐 命令行批处理示例
```bash
#!/bin/bash
# 批量处理文件夹中的所有图片

INPUT_DIR="./input"
OUTPUT_DIR="./output"

for img in "$INPUT_DIR"/*.png; do
    filename=$(basename "$img" .png)
    tesseract "$img" "$OUTPUT_DIR/$filename" \
        -l chi_sim+eng \
        --psm 6 \
        --oem 1 \
        -c user_defined_dpi=300 \
        -c lstm_choice_iterations=3 \
        -c beam_width=15
    echo "Processed: $filename"
done
```
